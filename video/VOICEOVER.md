# flightrec demo — voiceover script (video runs 2:57)

Read along while `video/flightrec-demo.mp4` plays. Timestamps match the Remotion
timeline (`video/remotion/src/timeline.ts`) exactly. Pace is ~135 wpm; each
block fits its slot with a beat to spare. Record audio however you like (phone
voice memo is fine), then either:

- mux it yourself:
  `ffmpeg -i video/flightrec-demo.mp4 -i voiceover.m4a -map 0:v -map 1:a -c:v copy -shortest video/flightrec-demo-voiced.mp4`
- or drop `voiceover.m4a` in `video/` and ask Claude to mux and verify it.

---

**[0:00 — title card]**
(quiet — let the card sit, take a breath)

**[0:07 — "Regulated firms can't deploy AI agents that act"]**
Regulated firms, banks, hospitals, can't deploy AI agents that actually do
things, because they can't prove afterwards what the agent did.

**[0:16 — "flightrec gives every agent a black box"]**
flightrec removes that blocker. Every action the agent takes is hash-chained
into an append-only black box.

**[0:24 — the /grant modal fills itself]**
Here's the agent in Slack. I ask for access through a structured form. Free
text is never an instruction, so a prompt injection in a message can't drive
it. Claude reasons about the request, but that's advisory. A deterministic
gate, bound to my own entitlements, is what decides.

**[0:48 — the green Access granted card]**
Granted: one hour of read on prod analytics. The request, the model's
reasoning, the decision, the action, and the token cost all just went into the
black box.

**[1:05 — the red Access denied card]**
Now the over-reach: admin. The model proposed GRANT, and it didn't matter. The
gate refused, because I'm not entitled to admin. The model can be wrong, or
jailbroken. The gate can't be talked out of policy.

**[1:23 — AUDIT VERIFIED card, report scrolling]**
One click gets the full audit, inside Slack. Every step is an event, every
event is hash-linked to the one before, and the verdict is recomputed from the
chain, never trusted from a stored value.

**[1:48 — "A log you can edit is worthless"]**
But here's the real test: what happens when someone edits the history
afterwards?

**[1:55 — the attacker's terminal types the UPDATE]**
An insider with database access drops the guard triggers and rewrites the
recorded event directly. No API, no permissions, straight into the file.

**[2:10 — the audit flips to INTEGRITY VIOLATION]**
The very next audit catches it. The chain breaks at exactly sequence four, the
event they touched. A log you can edit is worthless. This one makes every edit
undeniable.

**[2:25 — Claude Desktop + Slack split screen]**
And flightrec isn't just this bot. It's an MCP server. Here's Claude Desktop
recording a deploy into the same black box, and Slack auditing that run seconds
later. Any agent, one audit trail.

**[2:47 — close card]**
flightrec. The model proposes, the gate disposes, and the black box remembers
everything.

**[2:57 — end]**

---

Every number, hash, run id, and rationale on screen is real output from the
pipeline (generated by `video/gen_artifacts.mjs` + `mcp/scripts/gen_mcp.mjs`

(timing lives in `src/timeline.ts`).
